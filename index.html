Dependency mapping
via Copilot + Mermaid

A single Copilot prompt reads a SharePoint library of technical docs and a Teams meeting transcript, then generates Mermaid graph syntax that renders a live system dependency map — turning scattered documentation into an always-current architecture diagram.

Teams transcript SharePoint library GitHub Copilot Mermaid.js System dependencies Auto-generated diagram

How it works

  • Copilot is prompted to read a SharePoint library containing architecture docs, API specs, and integration runbooks
  • It also ingests the latest architecture review meeting transcript from Teams to capture any verbal decisions not yet in docs
  • A single prompt asks Copilot to return valid Mermaid graph syntax describing the system dependencies it found
  • The generated code is dropped into the Mermaid live editor — the diagram renders instantly, ready to share or embed

Why this matters

  • Auto-generated dependency maps become living technical documentation — fed directly into onboarding packs, runbooks, and requirements docs without anyone manually drawing a diagram
  • New engineers and business analysts can understand how systems connect in minutes rather than spending weeks extracting that knowledge from the people who built them
  • Requirements gathering accelerates when teams can see what a change touches before the conversation even starts — reducing the back-and-forth caused by undiscovered dependencies
  • Tribal knowledge locked in meeting transcripts and engineers' heads gets surfaced and codified automatically — so the organisation retains it even when people move on
